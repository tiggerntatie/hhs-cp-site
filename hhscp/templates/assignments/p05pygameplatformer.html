This project will give you experience writing graphical applications. You will also learn how to use classes and object-oriented programming to streamline your code.

What to do

Use Python to create a sandbox platform game. A platform game is a 2D game with a player that jumps between platforms (like Super Mario Bros). “Sandbox” simply means that instead of having structured levels, the user creates their own environment and plays around in it. In this case, the user will be able to create various objects at the mouse pointer by pressing keys on the keyboard. They will also be able to control a player object using the arrow keys.

The game will look something like this:

You must include the elements labeled in the image.

  1. Pressing 'w' creates a wall at the mouse. This is simply a square obstacle that blocks the movement of the player and other objects. I recommend making the walls snap to a grid so you can make smooth platforms.
  2. Pressing 'p' creates the player at the mouse (provided that there isn't already one on the screen). It is affected by gravity, and falls down until a wall stops it. The user can move the player by pressing and holding the left and right arrow keys. If the player is on the ground, the user can jump by pressing the up arrow key. These are some basic characteristics, but feel free to add extra controls for the user or interesting properties.
  3. Pressing 's' creates a spring at the mouse. If the player hits a spring, it will launch upwards (higher than a jump takes it). Springs are also affected by gravity so that, when created, they fall and come to rest on flat ground.

How to do it

Note: I recommend making a separate folder for this project, since you're going to end up with more than one .py file.

First, you'll want to create a main window to draw your game in. Download pygameapp.py and put it in your project's folder. This simple class is all you need to create a window with a working close button. You can do this by creating an instance of the class and calling its run() function, as follows: 

myapp = PygameApp()
myapp.run()

This won't get you a working game, however, and you are not allowed to change PygameApp. To add your own functionality to the class, create a new file for the game (I called mine PlatformGame.py) and import the PygameApp class. Then create a PlatformGame class that inherits from PygameApp. I won't guide you on what to do for this, but you'll probably want to override the poll() and handle_event() functions. Thanks to PygameApp, whatever code you put in the poll() function will run every tick of the clock (so if you set your app to run at 60 frames per second, this code will run 60 times each second). handle_event() is where mouse and keyboard input will come in; use this to detect keypresses and get the position of the mouse.

You should also make classes for all of the game objects (walls, the player, and springs). All of these objects (which we'll call actors) have some basic properties in common: Every actor has a position and a size. You'll also want every actor to be able to draw itself on the screen, check if it's overlapping another actor, etc.

I've made an Actor class that encapsulates all of this basic functionality. Download actor.py and put it in your project's folder. Open it and take a look at the code. Here are the basic functions of an actor:

  • __init__(): This initializes the actor. All actors have a position and a size, as well a shared list of all the actors in the game (more about this later).
  • __str__(): Represents this actor as a string. Because of this function, code such as print my_actor will show useful information instead of default text Python provides.
  • is_overlapping(): Checks if this actor is overlapping a particular other actor.
  • overlapping_actors(): Gets a list of all of the actors overlapping this actor.
  • draw(): Draws the actor to the window. By default, this draws the actor as a rectangle of color self.color, but by overriding this function you can give it any appearance you like.
  • timestep(): This function is meant to be called every frame of your application. It does nothing by default, but by overriding it, you can make your actor do something in response to the passage of time.
  • destroy(): Destroys the actor.

You may not modify the Actor class. Instead, create all of your game objects by making subclasses of Actor. Start off simple by creating a Wall class. Once you've done that, I suggest making another generic actor class, but this time for objects that are influenced by gravity. I called this class GravityActor.

You're probably wondering how to make all of these classes actually show up on the screen and do something. Remember how every actor wants a list of all the actors in the game? This is because keeping a list of actors is the best way (perhaps the only way) to keep track of them. In the poll() function in your game's main class, you can loop through all of the actors in your master list. First call its timestep() function to do anything specific to the actor, then draw it.

Extra ideas

There are lots of possibilities for extra features to add to your platform game.

  • A "platform" actor that serves as a floor but not a ceiling. The player can jump through the bottom of this platform, but will land on top of it.
  • A "laser turret" actor that fires laser bolts at the player.
  • Enemies that walk along the ground.
  • A jetpack activated by holding the down arrow key.
  • Your own ideas!
