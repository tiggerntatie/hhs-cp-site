Introduction

Computer Programming is a one-semester course that will introduce the basic techniques of computer programming using the Python programming language. Python is a modern, interpreted language that is relatively easy to learn, while possessing an astonishing depth and flexibility that make it suitable for sophisticated applications in business, arts, entertainment, science and engineering.

In the first few weeks of this class you will learn the tools and basic skills needed to write simple programs in Python. Each week will develop several new skills or techniques that may be required to complete a series of challenge assignments. Some of the skills you will learn during this period include:

  • Interactive mode: expressions, types, variables, collections, dir(), help()
  • Console I/O
  • Lambda functions
  • List comprehensions
  • Objects, properties, methods and inheritance
  • Conditionals and control flow statements
  • Functions

Later in the course you will plan and execute a series of one to two-week long mini-projects

I Provide You With:

  • Computers with Internet access.
  • Python 3.4
  • Python Integrated Development Environment: Eclipse Luna
  • Pygame Python game development library.
  • Use Google Drive to save your work.

If you want to work outside of class:

  • We will be using Python 3.4 during this course. Although this is the latest version that you will find available for download, you may notice that some people recommend using the older version: 2.7. Don't do it! The 2.x and 3.x versions are different enough that the examples in this course may not work correctly if you are using the older version. Python 3.4 has reached a level of maturity and usability (it was released almost six years ago!) that make it a great choice for introducing computer programming.
  • You may install the latest version of the Eclipse Luna on your personal computer. After installation use Help/Install New Software/ to install Pydev for Eclipse from http://pydev.org/updates (don't select Mylin integration). If you are installing to OSX then you will not be able to use the latest version of PyDev unless you upgrade Java to a newer version. If you do not upgrade Java then you must check the option to show all PyDev versions and select PyDev version 2.8.2 instead.
  • Pygame will be used later in the course, but is not required initially. Note: be careful to install the Pygame version that is correct for your operating system and Python 3.4. Check with me if you are having trouble finding the correct software (I have collected many files for installing to Windows and OSX).

I Expect YOU To:

  • Come to class.
  • Be on time. Please let me know if you expect to be late.
  • Be responsible. If you are absent on a day when work is due I will expect you to have it ready the day you return.
  • Always treat others with kindness and respect.

Textbook

There isn't one. I will have a couple of reference books on hand during class, but the best resources tend to be on the Internet. I expect you to use the Internet as a tool for solving problems and learning new techniques. Remember that most of the information you find about Python will be tailored for the 2.x version and either may not work or may not be the best approach for Python 3.x. Be aware!

X Periods

This class has an X period on Thursday. Please check in at the start of class (or make arrangements with me in advance). I am required to take and submit attendance for the X! The X is available to you for finishing your work, catching up or getting help.

Academic Honesty

All work submitted for credit must be your own. You may discuss the problems with others, but you must write your own solutions. For each problem, you must indicate who else you worked with or got any help from, small or big. Any written or online sources used (other than examples I have provided) must also be acknowledged; however, you may not consult any prior solutions to these challenges that you find on the Internet or elsewhere. You may undertake your final project with a partner, in which case your submitted work is expected to be yours or your partner's.

Classworks and Project Grading
Ongoing Challenges At the beginning of the semester, each week will bring a series of challenge assignments that you must complete and upload to a web site for automated verification.
Project Reviews Your final project will require planning. You or your team must develop a plan and demonstrate weekly progress towards completion. A well-designed and implemented application represents a substantial amount of work!
Mini Projects After the initial introductions you will work on some more substantial projects of your own choosing. For example, graphics programming or game development with Pygame. These will also be uploaded to the course web site, but will not be automatically verified.
Final Project This longer project may take up to a full quarter to complete. A final project presentation will serve as your final exam in the course.
Late Work All work is subject to being graded once the due date has arrived. Work that is completed or revised after the due date, may be awarded full credit within two weeks of the original due date. You must inform me by e-mail of updated work or any time you submit late work, as I have no way of knowing what you want me to look at!
Grading
Challenges 2 points for successful completion, 1 point for submission.
Projects 5 points for successful completion, with reductions for errors or missing functionality.
Progress Reports 2 points for final project progress reports
Final Project Final Project presentation is worth 20% of the semester grade.
